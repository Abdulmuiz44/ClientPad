About

ClientPad exists for businesses that need software to meet them where work already happens.

Many service businesses run on WhatsApp, spreadsheets, manual follow-ups, and scattered payment links. ClientPad turns that daily activity into open infrastructure developers can inspect, run, extend, and sell services around.

Open by default

The core stack is MIT licensed. Developers can self-host PostgreSQL, run migrations, create API keys, and build their own workflows without asking ClientPad for permission.

Commercial where it helps

ClientPad Cloud is the managed path for teams that want hosted infrastructure, operator auth, usage tracking, readiness diagnostics, billing checkout, and support.

What we are building

A practical infrastructure layer for small and growing businesses: WhatsApp lead capture, owner inboxes, pipelines, clients, API keys, usage analytics, and hosted Cloud operations.

ClientPad should be useful to a developer installing an npm package, a founder launching a client operations tool, and an operator checking whether real leads are flowing.
